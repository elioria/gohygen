# gohygen examples & real-world template compatibility

This document shows gohygen running **real hygen templates** — both the bundled
example generators and templates written for the original Node.js `hygen`.

Run the bundled demo:

```bash
go run ./examples
```

---

## 1. Bundled example generators (`examples/_templates`)

### `component` — the production [jondot/hygen-CRA](https://github.com/jondot/hygen-CRA) generator

This is the *unmodified* Create-React-App component generator from hygen's
author. It uses `h.inflection.undasherize`, three conditional component styles,
and a prepend-inject into a Storybook index.

```bash
gohygen component new --name user-card              # PureComponent (default)
gohygen component new --name nav-bar --functional   # function component
gohygen component new --name side-menu --stateful   # stateful Component
```

`user-card` → `src/components/user-card/index.js`:

```js
import React, { PureComponent } from 'react'
class UserCard extends PureComponent {
  render() {
    return <div>edit me: at components/user-card/index.js</div>
  }
}

export default UserCard
```

`--name user-card` becomes `UserCard` via `h.inflection.undasherize`, and the
default branch of the template's `<% if(locals.stateful) … %>` chain yields a
`PureComponent`. A companion `inject_story.t` prepends an import into
`src/stories/index.js`, guarded by `skip_if` so re-runs stay idempotent.

### `gomodel` — nested includes + idempotent inject

A Go model generator that demonstrates the headline feature: **deep nested
includes**. `model.go.ejs.t` includes `partials/header`, which includes
`partials/license`:

```
model.go.ejs.t ──include──▶ partials/header.ejs ──include──▶ partials/license.ejs
```

```bash
gohygen gomodel new --name blogPost
```

`models/blog_post.go`:

```go
// SPDX-License-Identifier: MIT
// Code generated by gohygen for "BlogPost". DO NOT EDIT.

package models

// BlogPost is a generated model.
type BlogPost struct {
	ID   int64  `json:"id"`
	Name string `json:"name"`
}

// TableName returns the database table for BlogPost.
func (BlogPost) TableName() string {
	return "blog_posts"
}
```

Both partials contribute their lines (license → header banner), and the helper
namespace `h` is available at every nesting level without being re-passed —
`h.changeCase.snakeCase("blogPost")` → `blog_post`,
`h.inflection.pluralize(...)` → `blog_posts`. A second template injects the new
type into `models/registry.go` after a `// gohygen:models` marker; running the
generator again is a no-op thanks to `skip_if`.

```go
var AllModels = []any{
	// gohygen:models
	&Comment{},
	&BlogPost{},
}
```

## 2. Compatibility with upstream hygen templates

`compat_test.go` runs **real templates vendored under `testdata/`** through
gohygen and checks the output:

### hygen's own metaverse corpus (`testdata/metaverse`)

Vendored from hygen's integration test suite. Covered generators:

| Generator | What it exercises |
|-----------|-------------------|
| `sample/new` | basic add |
| `worker/new` | `<% … %>` eval, `h.capitalize` |
| `inflection/new` | irregular plural in `to:` (`person` → `people.md`) |
| `conditional-rendering/new` | `to:` evaluating to `null` → file skipped |
| `positional-name/new` | positional argument fills `name` |
| `attrs-in-body/new` | `<%= attributes.to %>` (rendered frontmatter in body) |
| `mailer/new` | **multi-file + inject**, output checked **byte-for-byte against hygen's own `expected/` fixture** |
| `add-unless-exists/new` | `unless_exists` skip on re-run |

The mailer test is the strongest signal: the injected `html.ejs` (an
`!!js/regexp /You owe/` before-injection plus a `<%%= bill %%>` literal) matches
hygen's committed expected output exactly.

### Production CRA generators (`testdata/cra`)

The full [jondot/hygen-CRA](https://github.com/jondot/hygen-CRA) `component`
generator, testing the default / `--functional` / `--stateful` branches and
inject idempotency over repeated runs.

Run them:

```bash
go test -run 'TestMetaverse|TestCRA' -v ./...
```

All pass — gohygen renders templates authored for the Node.js tool unchanged.

## Notes on parity

- gohygen runs the template language and the ops; it does **not** execute
  JavaScript `prompt.js` / `index.js` hook files (those are a Node feature). For
  generators that compute extra locals in a hook, pass the equivalent values
  directly (e.g. as CLI flags or `Run` locals).
- `from:` shared templates, `.hygenignore`, subaction filters (`action:sub`),
  and `_default` actions are supported.
